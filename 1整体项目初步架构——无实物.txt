3. 关键模块模拟方案
3.1 虚拟米家设备
用一个 Python 类 VirtualHome 维护设备状态字典，并提供与控制逻辑完全相同的 API。

python
class VirtualHome:
    def __init__(self):
        self.devices = {
            "bedroom_light": {"status": "off", "brightness": 100},
            "living_curtain": {"status": "closed"},
            "water_heater": {"status": "off", "temperature": 40},
        }
    
    def execute(self, device_id, action, params=None):
        # 与真实米家 API 参数完全一致
        self.devices[device_id]["status"] = action
        # 打印日志或推送 WebSocket 更新到前端
        return {"success": True}
展示要点：每次设备状态变化，前端图标实时更新（灯亮/灭动画、窗帘开合效果）。

3.2 虚拟传感器
用时间驱动的数据发生器 + 手动注入模拟。

python
class VirtualSensorHub:
    def __init__(self):
        self.sensors = {
            "bedroom_door": "closed",
            "bedroom_motion": "no_person",
            "living_temp": 25.2,
            "phone_charging": True,
        }
    
    def get_state(self, sensor_name):
        return self.sensors[sensor_name]

    def simulate(self, scenario):
        # 场景注入：如 "morning_wakeup"
        if scenario == "morning_wakeup":
            self.sensors["bedroom_motion"] = "person_detected"
            self.sensors["bedroom_door"] = "open"
展示要点：提供场景按钮（“模拟起床”“模拟离家”等），一键触发传感器变化，测试主动提醒。

3.3 语音交互模拟
输入：网页文本框或命令行输入，代表用户的语音转写结果（跳过 ASR，直接给文本）。

输出：MiGPT 原本要发送给小爱 TTS 的文本，直接显示在界面消息区，并可选调用浏览器 TTS 朗读。

这样演示时，观众像和聊天机器人对话，但背后跑的是完整双引擎逻辑。

3.4 大模型接入
因为是演示，可不依赖私有化部署，灵活选择：

API 模式：接入通义千问/OpenAI/Claude API，真实展示大模型理解能力。

离线模拟：用小型本地模型（Ollama + Qwen2.5:7B）运行在演示机，确保无网可用。

录制回放（演示保底）：预录几段典型对话的模型响应，万无一失。

建议至少准备 录制回放 作为网络故障时的 fallback。

4. 演示流程设计（串联所有亮点）
以下是一个能展示所有核心特性的剧本，适合 10 分钟完整演示。

场景 A：本地快速响应（SOP 秒级执行）
用户在文本框输入：“打开卧室灯”。

路由层关键词命中，走本地 SOP，直接调用 VirtualHome.execute("bedroom_light", "on")。

界面回复：“好的，已打开卧室灯”，灯图标点亮。

延迟 < 0.2 秒，体现本地路径优势。

场景 B：复杂语义 + 设备组合
输入：“我今天累坏了，帮我营造一个好好休息的氛围”。

路由层判断复杂，送大模型。

大模型理解意图，返回结构化动作：关窗帘、开热水器、播放白噪音（TTS 模拟）。

虚拟设备依次变化，回复温柔安抚语。

同时，自学习观察器记录此次操作组合。

场景 C：自学习 SOP 养成演示
演示管理员后台，展示日志中出现 3 次“好累啊→关窗帘+开热水器+白噪音”。

点击“一键学习”或系统自动触发学习。

本地 SOP 库新增条目。

再次输入：“好累啊”，这次直接走本地 SOP，不调大模型，响应瞬间完成。

展示 SOP 管理列表，可禁用/删除。

场景 D：渐进式记忆与主动情境提醒
提前通过对话注入记忆：“我每天早上 7:50 要出门上班，通勤 30 分钟”。
（知识图谱存入 (user, commute_time, 30min), (user, leave_time, 07:50)）

设置虚拟时间 7:10，传感器仍显示“卧室有人、窗帘关闭”。

情境引擎计算：leave_time - commute_time - buffer(10min) = 7:10，触发提醒。

界面主动弹出消息（并 TTS 语音）：“主人，现在 7:10 了，按您 30 分钟通勤时间，差不多该出发了哦，需要我拉开窗帘吗？”

用户回复“好的”，设备联动执行。

展示记忆数据页面，查看系统自动记录的信息。

场景 E：隐私与用户控制
用户说“关闭智能提醒”。

系统立刻暂停主动情境触发，界面提示“主动提醒已暂停”。

展示隐私面板，一键删除所有记忆、SOP 或传感器历史。
