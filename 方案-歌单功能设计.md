# 歌单播放功能设计文档

## 1. 需求概述

| 需求 | 说明 |
|------|------|
| 歌单目录 | 用户自行在 `music/` 下创建子文件夹作为歌单（如 `轻音乐/`、`收藏/`） |
| 语义匹配 | 小智根据用户语境/情景自动选择匹配的歌单（学习→轻音乐、放松→收藏） |
| 歌单播放 | 播放歌单时依次播放文件夹内所有歌曲（自动连播） |
| 显示优先级 | 歌单名称 > 当前曲目名（播放器优先显示"轻音乐"再显示当前曲名） |

## 2. 存放结构

```
frontend/public/music/
├── running-up-that-hill.mp3          ← 单曲（现有）
├── playlists/                        ← 歌单目录（新建）
│   ├── 轻音乐/                       ← 歌单名 = 文件夹名
│   │   ├── track1.mp3
│   │   └── track2.mp3
│   ├── 学习助眠/
│   │   └── ...
│   └── 收藏/
│       └── ...
└── *.mp3                             ← 普通单曲（现有）
```

> **为什么用子文件夹而不是数据库？** 用户可以直接拖放 MP3 文件到文件夹，零配置新增歌单。

## 3. 后端改动

### 3.1 `playlist_service.py` — 歌单扫描

新增两个函数：

```python
# 歌单索引缓存
_playlist_index: dict[str, list[dict]] = {}    # 歌单名 → 歌曲列表

def refresh_playlists():
    """扫描 music/playlists/ 下的子文件夹作为歌单"""
    playlists_dir = os.path.join(MUSIC_DIR, "playlists")
    if not os.path.isdir(playlists_dir):
        os.makedirs(playlists_dir, exist_ok=True)
    _playlist_index.clear()
    for folder in sorted(os.listdir(playlists_dir)):
        folder_path = os.path.join(playlists_dir, folder)
        if not os.path.isdir(folder_path):
            continue
        songs = []
        for f in sorted(os.listdir(folder_path)):
            if f.endswith((".mp3", ".wav", ".flac", ".ogg")):
                info = _parse_filename(f)
                info["url"] = f"/music/playlists/{folder}/{f}"
                songs.append(info)
        if songs:
            _playlist_index[folder] = songs
    logger.info(f"歌单已刷新: {len(_playlist_index)} 个, 共 {sum(len(v) for v in _playlist_index.values())} 首")

def get_playlist(name: str) -> list[dict] | None:
    """按歌单名获取歌曲列表（含 url 路径）"""
    if not _playlist_index:
        refresh_playlists()
    return _playlist_index.get(name)

def list_playlists() -> dict[str, int]:
    """返回 {歌单名: 歌曲数}"""
    if not _playlist_index:
        refresh_playlists()
    return {k: len(v) for k, v in _playlist_index.items()}
```

### 3.2 `main.py` — 音乐控制消息加 `playlist_name`

`_send_music_control` 函数中，当 `music_action` 携带 `playlist` 字段时：
1. 调用 `get_playlist(playlist_name)` 获取歌单歌曲
2. 发送 `music_control` 消息，包含：
   - `action: "play"`
   - `playlist_name` — 歌单名（前端显示用）
   - `songs` — 歌单内所有歌曲
   - `song_name` — 第一首歌名（可选）

消息格式扩展：
```python
ws.send_json({
    "type": "music_control",
    "action": "play",
    "playlist_name": "轻音乐",     # ← 新增
    "song_name": "track1",
    "songs": [song1, song2, ...],
})
```

### 3.3 语义匹配 — LLM 输出歌单名

**系统提示词**（`llm_service.py`）补充一条：

```
10. 【歌单匹配 — 重要】根据用户语境自动匹配歌单名输出到 ACTIONS。
    如果用户的请求涉及特定场景/情绪，用 playlist 字段指定歌单名：
    - 学习/看书/做作业 → "轻音乐"
    - 放松/休息 → "收藏"
    - 睡觉/冥想 → "助眠"
    - 工作/专注 → "专注"
    输出格式：
    [ACTIONS]{"music":{"action":"play","playlist":"轻音乐","query":""}}[/ACTIONS]
    如果用户指定了具体歌名，不要用 playlist，用 query。
```

## 7. 向后兼容

- 不加 `playlist_name` 时，播放器行为不变（显示曲名）
- 歌单目录 `playlists/` 不存在时自动创建，不影响现有单曲
- `get_all_songs()` 仍然返回根目录的单曲，不影响旧功能

## 8. 已确定的交互设计

| 决策 | 结论 |
|------|------|
| 播放顺序 | **随机** — 每次启动歌单时随机打乱顺序 |
| 播完循环 | **循环** — 播完自动从头（重新打乱）再播 |
| 创建方式 | **文件夹 + UI** — 支持手动拖放，前端播放器也提供创建歌单/移动歌曲的功能 |
| 歌单封面 | 暂不实现，后期可加 |



## 9. 切歌问题修复

### 根因分析
切歌时 xiaoai_execute 立即返回好的，已切换到下一首文本回复（走 token/done 路径），但实际切歌动作依赖 **\`_send_music_control\` 发送 music_control: next 到前端**。旧代码在发送前检查 **\`_search_cache\`**（仅搜索流水线写入），歌单播放路径没有写入该缓存，导致切歌指令被拦截。

### 修复内容
| 文件 | 改动 |
|------|------|
| \`backend/main.py\` | **去掉 \`_search_cache\` 检查**，next/prev 始终直接发送到前端。前端已有歌曲队列就切，没有则无害通过 |
| \`frontend/src/hooks/useMusicPlayer.ts\` | **next/prev 队列为空时不播默认曲目**，保持静音而不是突然放歌 |

### 切歌完整链路（修复后）
```
用户:  切歌
后端:  classify → xiaoai 路径
       xiaoai_execute → reply=好的，已切换到下一首
       → ws.send token+done（前端显示文本）✅
       → _maybe_tts（音箱朗读）✅
       → _send_music_control(ws, {action:next})
         → 直接发送 music_control: next ✅
前端:  handleMusicControl({action:next})
       → next() → queueRef.current 有歌 → playIndex(idx+1) ✅
       没歌 → 无害通过（不播默认曲目）
```
